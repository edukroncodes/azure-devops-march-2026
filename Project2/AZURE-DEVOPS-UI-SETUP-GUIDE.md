## 11. App Service application settings

Configure in Portal for **each** Web App after creation.

### 11.1 API app (`ecommerce-api-dev-{suffix}` / prod equivalent)

| App setting name | Example value |
|------------------|---------------|
| `ASPNETCORE_ENVIRONMENT` | `Development` (dev) or `Production` (prod) |
| `ConnectionStrings__DefaultConnection` | SQL connection string from Portal → SQL database → Connection strings |
| `Jwt__Key` | Same as Bicep `jwtKey` (32+ chars) |
| `Jwt__Issuer` | `ECommerce.Api` |
| `Jwt__Audience` | `ECommerce.Clients` |
| `Jwt__ExpiryHours` | `24` |
| `AllowedOrigins__0` | `https://{webAppName}.azurewebsites.net` |
| `APPINSIGHTS_INSTRUMENTATIONKEY` | From Application Insights → `ecommerce-ai-dev` |

**UI path:** App Service → **Environment variables** (or Configuration → Application settings)

### 11.2 Web app (`ecommerce-web-dev-{suffix}` / prod equivalent)

## 13. 20 pipeline stages reference

| # | Stage display name | What it does | Azure resource needed |
|---|-------------------|--------------|------------------------|
| 01 | Validate Repository | Checks files exist | None |
| 02 | Restore Dependencies | NuGet restore | None |
| 03 | SonarQube Prepare | Installs SonarScanner | SonarCloud (optional) |
| 04 | Build Solution | `dotnet build` | None |
| 05 | Unit Tests | Runs xUnit tests | None |
| 06 | Code Coverage | Coverage + publish | None |
| 07 | SonarQube Analyze | Sonar scan | `SONAR_TOKEN` (optional) |
| 08 | SonarQube Quality Gate | Summary | None |
| 09 | NuGet Vulnerability Audit | `dotnet list --vulnerable` | None |
| 10 | Trivy Filesystem Scan | Vuln scan | None |
| 11 | Trivy Misconfiguration Scan | IaC/config scan | None |
| 12 | Trivy Secret Scan | Secret scan | None |
| 13 | .NET Format Check | `dotnet format` | None |
| 14 | Publish API | Zip artifact `api-package` | None |
| 15 | Publish Web | Zip artifact `web-package` | None |
| 16 | Trivy Artifact Scan | Scan zips | None |
| 17 | Generate SBOM | CycloneDX + SPDX | None |
| 18 | Package Build Artifacts | Publishes `drop` artifact | None |
| 19 | Deploy Development | Deploy zips to dev apps | Web Apps + `HealthcareECommerce-Azure` |
| 20 | Deploy Production | Deploy zips to prod apps | Web Apps + `HealthcareECommerce-Azure` |

### Pipeline artifacts produced

| Artifact name | Contents |
|---------------|----------|
| `security-reports` | NuGet audit text |
| `trivy-fs` | Trivy filesystem reports |
| `trivy-config` | Trivy config reports |
| `trivy-secrets` | Trivy secret reports |
| `trivy-artifacts` | Trivy scan on zips |
| `api-package` | Published API `.zip` |
| `web-package` | Published Web `.zip` |
| `sbom` | SBOM JSON files |
| `drop` | Combined `api-package`, `web-package`, `sbom` |

## 14. Checklist before first run

### Azure DevOps (UI)

- [ ] Organization exists: `https://dev.azure.com/kcskypoint`
- [ ] Project created: `HealthcareECommerce`
- [ ] Repository `HealthcareECommerce` contains code + `azure-pipelines.yml`
- [ ] Branches `main` and `develop` exist
- [ ] Pipeline `HealthcareECommerce-CI` points to `/azure-pipelines.yml`
- [ ] Service connection `HealthcareECommerce-Azure` created and authorized for all pipelines
- [ ] All [pipeline variables](#6-pipeline-variables-all-names) added
- [ ] Optional: `SONAR_TOKEN` secret set
- [ ] Optional: environments `ecommerce-dev`, `ecommerce-prod` created
- [ ] Optional: SonarCloud project `HealthcareECommerce` + service connection

### Azure Portal (UI)

- [ ] App Service quota ≥ 1 VM (if doing live deploy)
- [ ] Resource group `rg-ecommerce-apps` created
- [ ] Bicep deployed for **dev** (`environment=dev`)
- [ ] Bicep deployed for **prod** (`environment=prod`) — if using separate prod apps
- [ ] Pipeline variables `apiAppNameDev`, `webAppNameDev`, `apiAppNameProd`, `webAppNameProd` updated from deployment **Outputs**
- [ ] App Service application settings configured on all four apps
- [ ] SQL firewall allows Azure services

### First pipeline run

- [ ] Pipelines → `HealthcareECommerce-CI` → **Run pipeline** → branch `main`
- [ ] Confirm all 20 stages green (19–20 validate packages; live deploy after apps exist)
